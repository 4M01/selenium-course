XPath Examples

What is XPath?

XPath is a query language for selecting nodes from an XML/HTML document. It provides a way to navigate through elements and attributes in a document.

Why use XPath? XPath provides powerful flexibility when other locators like ID, name, or CSS selectors aren't available. It allows navigation through the DOM tree in multiple directions and can use text content to identify elements.

Absolute vs Relative XPath

Absolute XPath

Starts with single slash (/) - begins from root

/html/body/div/div[2]/input

Disadvantages:
  • Changes if DOM structure changes
  • Long and complex
  • Not maintainable

Relative XPath

Starts with double slash (//) - can start from anywhere

//input[@id='username']

Advantages:
  • More resilient to DOM changes
  • Shorter and readable
  • Maintainable

When to Prefer Custom XPaths

Prefer custom XPaths when:
  • Elements have no unique ID, name, or class attributes
  • You need to locate elements based on text content
  • You need to locate elements based on their relationships to other elements
  • Dealing with dynamic IDs that change on page refresh
  • Need to locate an element based on multiple conditions
